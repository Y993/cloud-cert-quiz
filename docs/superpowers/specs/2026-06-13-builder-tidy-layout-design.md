# 構成図ビルダー「整列」ボタン — 設計

作成日: 2026-06-13 / 対象: CLOUDCERT_ 構成図ビルダー（builder.html, AWS/GCP/Azure 共通）

## 目的

ユーザーがドラッグで雑に並べた構成図を、ボタン一発で**接続の流れに沿ったきれいなレイアウト**へ自動整列する。
学習ツールとして「正しい配置」を壊さないことが最優先 — 枠（VPC 等）の所属判定とサイズは常に尊重する。

## 採用方式

フロー配置（左→右の階層レイアウト）。**入れ子対応の階層フロー配置（方式B）** を採用。
外部ライブラリ（dagre/elk）は使わない（本プロジェクトはビルド無しの素の JS 方針）。

## 不変条件（壊してはいけないこと）

- **枠の所属が保たれる**: 整列前にあるノードが属していた枠（および枠の入れ子関係）は、整列後も同じ所属を保つ。
  - 検証軸: 同梱テンプレートを整列したあと `validate()` の指摘がゼロのまま（既存の黄金律テストと同じ思想）。
- ノード・枠・接続の**集合は不変**（追加・削除しない。座標とサイズだけ変える）。
- Undo（Ctrl+Z）で整列前に戻せる。自動保存・PNG 書き出し・共有リンクと整合（state を書き換えるだけ）。

## アルゴリズム（`tidyLayout(arch, state)`）

DOM 非依存の純関数として `js/builder-core.js` に追加（`validate` / `membership` と同じ作法）。
`state` を破壊的に更新し、`state` を返す。

### 1. 所属ツリーの構築（現在の座標から）

- 各ノードの親枠 = 中心を含む最小面積の枠（既存 `membership` の最内＝chain 末尾）。
- 各枠の親枠 = 自分を除き、自分の中心を含む最小面積の枠。
- 親を持たない要素 = ルート直下。
- これで「ルート ▸ aws ▸ vpc ▸ public/private ▸ ノード」のような木ができる。
  region レベルのノード（route53・cloudfront 等）は vpc に入らず aws 直下の子になる。

### 2. コンテナ単位の再帰レイアウト（葉→根）

`layoutContainer(children, edges)` を再帰で適用し、各要素の**相対座標とブロックサイズ**を確定する。

- 子要素 = その階層の直下ノード（サイズ `NODE_W×NODE_H`）＋直下の枠（サイズは再帰結果）。
- **edge の引き上げ**: 各 edge の端点を「この階層の直下の子」に解決（自分自身か、自分を含む直下の子枠）。
  両端が別々の直下の子に解決できたときだけ、その子同士の局所有向辺として採用。
- **層割り当て（最長経路法・左→右）**: 先行ノードが無い要素＝層0、それ以外は `layer(v)=max(layer(pred)+1)`。
  循環は進行中ノードを層0として打ち切り（back-edge を無視）。線が一切ない孤立要素は末尾の層へ。
- **層内の並び順**: 現在の交差軸座標（左→右配置なら現在の y 中心）で安定ソート。ユーザーの頭の中の上下関係を保つ。
- **配置**: 層 = 列。列の x = それ以前の列幅の累積 + `COL_GAP`。列内は縦に `ROW_GAP` 間隔で積み、列ごとにブロック中央へ縦センタリング。
- ブロックサイズ = 全列の合計幅 / 最大列高。

### 3. 枠のフィット

枠は「内側ブロック＋パディング」をサイズにする（上はラベル分を広め: `PAD_TOP`、左右下: `PAD`）。
枠の子は枠位置 +（左パディング, 上パディング）に再帰配置。これにより枠は常に中身を内包する。

### 4. 絶対座標の確定

ルートを `(MARGIN, MARGIN)` に置き、ブロックの `place(x, y)` を再帰的に呼んで
node.x/y・group.x/y/w/h を確定。最後に各座標を 0 以上にクランプ（キャンバス上限超過は許容、`zoomFit` で全体表示）。

### チューニング定数（1か所にまとめる）

- 配置方向: 左→右（`COL` = x 軸 / 層、`ROW` = y 軸 / 層内）。上→下に変えるのは軸の入れ替え1か所。
- `COL_GAP`≈60, `ROW_GAP`≈30, `PAD`≈22, `PAD_TOP`≈34, `MARGIN`≈60。

## UI 結線（builder.js / builder.html / builder.css）

- `builder.html` のツールバーに `<button class="tb-btn" id="btnTidy" title="接続の流れに沿ってレイアウトを自動整列">整列</button>` を追加（ズーム群の隣）。
- `builder.js`: `btnTidy` の click で
  `if (!state.nodes.length) return;` →（あれば）`pushHist()` → `CORE.tidyLayout(ARCH, state)` →
  `sel=null; render(); save(); zoomFit();` → `flashSave("レイアウトを整えました ✓")`。
- ノードが0個のときは何もしない（または「整える対象がありません」を表示）。
- CSS は既存 `.tb-btn` を流用（新規スタイル不要）。

## テスト（`tools/test/builder-core.test.js` に追加）

1. **層順**: `user→alb→ec2` のような直列 state を整列すると x が単調増加（左→右）。
2. **枠の包含**: 枠＋中身の state を整列後、各ノード中心が自分の親枠の矩形内にある。
3. **非重複**: 同一コンテナ直下のノード同士の矩形が重ならない。
4. **黄金律（回帰）**: 全同梱テンプレートを `tidyLayout` → `validate` で指摘ゼロのまま。
5. **集合不変**: 整列前後で nodes/groups/edges の個数と uid 集合が不変。
6. **空入力**: nodes 空でも例外を出さない。

## スコープ外（YAGNI）

- 交差最小化（バリセンタ法の本格実装）。v1 は現在座標順を保つだけ。
- 上→下レイアウトの UI 切替（定数で対応可能だが UI は出さない）。
- アニメーション付き移動。
