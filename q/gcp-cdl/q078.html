ある企業が、Google Cloudの「Cloud Load Balancing」の主な役割を検討しています。Cloud Load Balancingの説明として最も正確なものはどれですか？

  1. A. Cloud Load BalancingはVMのCPU使用率を監視するモニタリングツールである
  2. B. Cloud Load BalancingはDNS名前解決を管理するサービスである
  3. C. Cloud Load Balancingは受信トラフィックを複数のバックエンドインスタンスに分散させることで、スケーラビリティと高可用性を提供するサービスである
  4. D. Cloud Load BalancingはCloud StorageとBigQuery間のデータ転送を管理するサービスである
解答と解説を見る

正解: C

Cloud Load Balancingは受信HTTPSリクエストやTCPトラフィックを複数のバックエンド（VMインスタンス、GKEノード等）に分散させることで、単一インスタンスへの負荷集中を防ぎ、スケーラビリティと高可用性を実現する。グローバルHTTPS LBはAnyCast IPにより世界中の最も近いエッジから処理できる。CPU監視はCloud Monitoring。Cloud StorageとBigQuery間のデータ転送は管理しない。DNS管理はCloud DNS。
