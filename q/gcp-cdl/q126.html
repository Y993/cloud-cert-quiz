CDLデータイノベーションとAIMEDIUM単一選択

ある企業が、リアルタイムの株価データと過去の取引履歴を組み合わせてMLモデルの推論に使いたいと考えています。モデルが「今日の株価（リアルタイム）」と「過去3年間の履歴（バッチ）」の両方の特徴量を同時に参照できるようにするために最適なGoogle Cloudのサービスはどれですか？

  1. A. Vertex AI Feature Store（オンライン特徴量とオフライン特徴量の両方を統一的に管理・配信）
  2. B. Cloud Scheduler（定期バッチ実行）
  3. C. Cloud Pub/Sub（メッセージキュー）
  4. D. Cloud Bigtable（リアルタイムデータのみ対応）
解答と解説を見る

正解: A

Vertex AI Feature Storeはオンライン（低レイテンシでのリアルタイム特徴量配信）とオフライン（大規模バッチ学習用の特徴量取得）の両方をサポートする特徴量ストア。リアルタイムの株価データはオンライン配信で推論時に取得し、過去の取引履歴はオフラインで学習時に取得するという用途に対応できる。Cloud Bigtableはリアルタイム読み書きには優れるが特徴量管理の統合機能はない。Cloud Schedulerはバッチジョブのスケジューリングのみ。Cloud Pub/Subはメッセージングサービスで特徴量ストアではない。
