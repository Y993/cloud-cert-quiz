CDLインフラストラクチャとアプリケーションのモダナイゼーションMEDIUM単一選択

ある企業が、短時間で完了するデータ変換処理（画像リサイズ、JSONパース等）を実装したいと考えています。サーバーのプロビジョニングや管理を一切行わず、コードの実行時間だけに課金されるサービスとして最も適切なものはどれですか？

  1. A. Cloud Functions
  2. B. Google Kubernetes Engine
  3. C. Compute Engine
  4. D. Cloud Dataproc
解答と解説を見る

正解: A

Cloud FunctionsはGoogleのサーバーレスFaaS（Function as a Service）であり、インフラ管理不要でコードのみをデプロイし、イベント（HTTP、Pub/Subメッセージ等）をトリガーに実行される。実行時間（100msずつ）と呼び出し回数のみに課金されるため、短時間処理の場合は非常にコスト効率が高い。Compute Engineは仮想マシンでインフラ管理が必要。GKEはコンテナオーケストレーションで管理オーバーヘッドがある。Cloud DataprocはSpark/Hadoopクラスターで大規模バッチ処理向き。
