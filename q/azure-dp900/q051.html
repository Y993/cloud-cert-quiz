DP-900Azure のリレーショナル データMEDIUM単一選択

ある企業が、オンプレミスで稼働している SQL Server アプリケーション（SQL Server Agent ジョブ・リンクサーバー・CLR 統合を使用）を Azure に移行したいと考えている。コードや設定の変更を最小限に抑えつつ、データベースエンジンの管理負荷も軽減したい。最も適切なサービスはどれか。

  1. A. Azure Database for PostgreSQL：オープンソースのマネージドリレーショナルサービス
  2. B. Azure SQL Database：最新の PaaS サービスだが一部の SQL Server 機能が非対応
  3. C. Azure Virtual Machines 上の SQL Server：オンプレミスと同等の環境を IaaS で再現
  4. D. Azure SQL Managed Instance：SQL Server との近完全互換を持つマネージド PaaS サービス
解答と解説を見る

正解: D

Azure SQL Managed Instance は SQL Server とのほぼ完全な互換性を持つマネージドサービスであり、SQL Server Agent・リンクサーバー・CLR 統合・データベースメール・Service Broker など Azure SQL Database では利用できない機能をサポートする。オンプレミス SQL Server アプリを最小限の変更で移行しながら、OS・パッチ適用・バックアップの管理から解放されるため最適な選択肢である。Azure SQL Database は SQL Server Agent などの機能が非対応であり、コード変更が必要になる可能性が高い。Azure VM 上の SQL Server は互換性が高いが OS を含む全管理がユーザー負担となる。Azure Database for PostgreSQL は PostgreSQL エンジンであり、SQL Server アプリとは互換性がない。
