DP-900Azure の分析ワークロードMEDIUM単一選択

ある企業が Azure Stream Analytics のジョブで、Twitter（X）から受信したツイートデータを処理し、商品名のキーワード別にツイート数をリアルタイムで集計したい。入力として Azure Event Hubs を使用し、結果を Power BI のリアルタイムダッシュボードに出力したい。この構成で Azure Stream Analytics の「出力」として設定すべきものはどれか。

  1. A. Azure SQL Database：集計結果をテーブルに書き込む
  2. B. Power BI：リアルタイムデータセットとして Power BI サービスに直接出力する
  3. C. Azure Blob Storage：結果データをファイルとして保存する
  4. D. Azure Service Bus：集計結果をキューメッセージとして送信する
解答と解説を見る

正解: B

Azure Stream Analytics は Power BI を出力シンクとしてネイティブにサポートしており、集計結果を Power BI の「ストリーミングデータセット」としてリアルタイムで送信できる。Power BI ダッシュボードはこのストリーミングデータセットを参照するタイルを設定することで、データが届くたびに自動更新されるリアルタイムダッシュボードを実現できる。選択肢 C の Blob Storage はデータをファイルとして保存する方式であり、Power BI ダッシュボードのリアルタイム更新には直接使えない。選択肢 A の Azure SQL Database への書き込みも可能だが、Power BI ダッシュボードのリアルタイム表示には SQL Database よりもストリーミングデータセットの方がレイテンシが低い。選択肢 D の Service Bus はメッセージング向けであり、Power BI への直接出力ではない。
