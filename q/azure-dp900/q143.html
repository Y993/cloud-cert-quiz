DP-900Azure の分析ワークロードHARD複数選択

ある企業が Azure Stream Analytics でリアルタイムストリーミング分析を設計している。Stream Analytics のウィンドウ関数について正しい説明を2つ選択してください。

  1. A. スナップショットウィンドウ（Snapshot Window）はウィンドウサイズではなく同一のパーティションキーを持つイベントをグループ化する
  2. B. スライディングウィンドウ（Sliding Window）は新しいイベントが到着するたびに境界を移動させるため、隣接する複数のウィンドウにイベントが含まれる
  3. C. タンブリングウィンドウ（Tumbling Window）は重複しない固定サイズの時間枠であり、各イベントは必ず1つのウィンドウにのみ属する
  4. D. ホッピングウィンドウ（Hopping Window）はウィンドウサイズとホップサイズが常に一致するため、タンブリングウィンドウと同一である
  5. E. セッションウィンドウ（Session Window）は固定サイズで定義されるウィンドウであり、イベントの到着に関係なく一定間隔でデータを集計する
解答と解説を見る

正解: B, C

タンブリングウィンドウは重複しない固定サイズの時間枠であり（例: 10秒ごと）、各イベントは必ず1つのウィンドウにのみ属する（選択肢 C が正解）。スライディングウィンドウは新しいイベントが到着するたびにウィンドウの終端が移動するため、時間的に隣接するウィンドウにイベントが重複して含まれる（選択肢 B が正解）。選択肢 D は誤りで、ホッピングウィンドウはウィンドウサイズとホップサイズを独立して設定できるため（例: 10秒ウィンドウを5秒ごとにホップ）、タンブリングウィンドウと同一ではない。ホップサイズがウィンドウサイズと等しい場合のみタンブリングウィンドウと同等になる。選択肢 E は誤りで、セッションウィンドウは一定時間内にイベントが発生しない（ギャップが生じた）タイミングでウィンドウが閉じるアクティビティベースのウィンドウであり、固定サイズではない。選択肢 A は誤りで、スナップショットウィンドウは同一のパーティションキー（PARTITION BY の値）を持つ同時刻イベントをグループ化するウィンドウである。
