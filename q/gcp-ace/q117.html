Cloud SQL for PostgreSQL インスタンスを作成するコマンドとして正しいものはどれですか？

  1. A. gcloud databases create my-db --type=postgresql --version=15 --region=asia-northeast1
  2. B. gcloud sql create instance my-db --database=postgres --tier=db-f1-micro
  3. C. gcloud cloud-sql instances create my-db --engine=postgres15 --region=asia-northeast1
  4. D. gcloud sql instances create my-db --database-version=POSTGRES_15 --tier=db-f1-micro --region=asia-northeast1
解答と解説を見る

正解: D

Cloud SQL インスタンスの作成コマンドは gcloud sql instances create <NAME> で、--database-version で POSTGRES_15 などのバージョンを指定し、--tier でマシンタイプ（db-f1-micro、db-n1-standard-2 など）を指定します。--region でリージョンを指定します。gcloud databases create は存在しないコマンドです。gcloud sql create instance は引数の順序が誤りです（正しくは gcloud sql instances create）。gcloud cloud-sql は存在しないサブコマンドグループです（gcloud sql が正しいグループです）。
