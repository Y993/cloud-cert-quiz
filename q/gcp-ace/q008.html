ACEクラウドソリューション環境の設定MEDIUM単一選択

あなたの会社は Google Cloud プロジェクトの費用増加を監視したいと考えています。特定のプロジェクトの予算の 80% を超えた時点で、Pub/Sub トピックへの通知を自動的に送信したい場合、どの設定が必要ですか？

  1. A. Cloud Monitoring のアラートポリシーを設定し、請求メトリクスに閾値を設定する
  2. B. Cloud Scheduler で定期的に予算チェックジョブを実行し、Pub/Sub に通知する
  3. C. Cloud Billing の予算に Pub/Sub トピックを通知チャネルとして設定する
  4. D. Cloud Functions で請求 API を定期実行し、80% 超過時に Pub/Sub に publish する
解答と解説を見る

正解: C

Cloud Billing の予算機能には、しきい値超過時に Pub/Sub トピックへ自動通知するネイティブ機能があります。予算設定画面で「Pub/Sub トピックに通知する」を有効にし、接続するトピックを指定するだけで設定完了です。これにより Cloud Functions や他の自動化ツールと連携したコスト制御の自動化が可能になります。Cloud Monitoring は課金メトリクスをサポートしますが、Pub/Sub への直接通知設定は予算アラートほどシンプルではありません。Cloud Functions や Cloud Scheduler を使ったポーリング方式は実装コストが高く、ネイティブ機能で代替できます。
