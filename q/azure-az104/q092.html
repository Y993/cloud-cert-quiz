AZ-104Azure コンピューティング リソースのデプロイと管理HARD複数選択

ある企業がVM拡張機能を使ってAzure VM群の運用自動化を進めている。VM拡張機能（VM Extensions）について正しい説明を2つ選択してください。

  1. A. VM拡張機能はWindowsVMにのみ対応しており、LinuxVMには使用できない
  2. B. VM拡張機能はARMテンプレート・BicepのリソースとしてVMデプロイと同時に宣言でき、デプロイ後のVM設定を Infrastructure as Codeで管理できる
  3. C. Custom Script Extensionは1つのVMに対して同じ拡張機能名で複数のスクリプトを同時に実行できる
  4. D. VM拡張機能はVM作成後にインストールすることはできず、必ずVM作成時のARMテンプレートで指定しなければならない
  5. E. 同一VMに複数の拡張機能（例: Custom Script Extension、Azure Monitor Agent、Microsoft Antimalware）を同時にインストールして並行稼働させることができる
解答と解説を見る

正解: B, E

VM拡張機能について正しい説明は2つある。第一に、VM拡張機能はARMテンプレートまたはBicepでMicrosoft.Compute/virtualMachines/extensionsタイプのリソースとして宣言できる。VMと同一テンプレート内でdependsOnを使ってVM作成後に適用するよう記述でき、インフラのコード化（IaC）が可能。第二に、同一VMに複数の拡張機能を同時にインストールして稼働させることができる。例えばAzure Monitor Agent（メトリクス/ログ収集）、Microsoft Antimalware（ウイルス対策）、Custom Script Extension（スクリプト実行）などを同一VM上で並行稼働させる構成は一般的。選択肢DはVM作成後にポータル・CLI（az vm extension set）・PowerShellでも拡張機能をインストール可能なため誤り。選択肢AはLinuxVMにも多数の拡張機能（Azure Monitor Agent Linux版、Custom Script Extension for Linux等）が対応しているため誤り。選択肢CはCustom Script Extensionは同一VMに対して一時点で1インスタンスしか実行できず、同じ名前の拡張機能を複数同時にインストールはできない（再実行は再インストールまたは削除→再追加が必要）。
