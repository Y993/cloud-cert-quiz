AZ-104Azure コンピューティング リソースのデプロイと管理EASY単一選択

ある企業が Azure 上に新しい仮想マシン（VM）を作成しようとしている。運用チームは、VM のハードウェア障害に備えて複数のVMを物理的に異なるサーバーラックに分散配置し、データセンター内でのラック単位障害に対して99.95%のSLAを確保したいと考えている。最小の構成変更でこの要件を満たすには何を使用すべきか。

  1. A. 仮想マシン スケール セット（VMSS）：需要に応じてVMを自動増減させる
  2. B. 可用性セット（Availability Sets）：更新ドメインと障害ドメインを使って同一データセンター内でVMを分散する
  3. C. 可用性ゾーン（Availability Zones）：異なるデータセンター間でVMを分散配置する
  4. D. 近接配置グループ（Proximity Placement Groups）：複数VMを物理的に近接させてレイテンシを最小化する
解答と解説を見る

正解: B

可用性セット（Availability Sets）は、同一データセンター内で複数のVMを障害ドメイン（最大3つの独立した電源・ネットワークのラック群）と更新ドメイン（最大20個、順次再起動される論理グループ）に自動分散し、ハードウェア障害や計画的メンテナンスによるダウンタイムを最小化する。2台以上のVMを可用性セットに配置することで99.95%のSLAが得られる。選択肢Cの可用性ゾーンは異なる物理データセンター間での分散（99.99% SLA）であり、ラック単位障害への対処としては過剰でコストも高い。選択肢Dの近接配置グループはレイテンシ最小化が目的であり、冗長性は提供しない。選択肢AのVMSSはスケーリングが主目的であり、可用性確保の仕組みは別途設定が必要。
