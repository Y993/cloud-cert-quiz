AZ-104Azure コンピューティング リソースのデプロイと管理MEDIUM単一選択

ある企業のインフラチームがAzure VMのデータディスク（Managed Disk）のバックアップ方法を検討している。特定時点のディスク全体のコピーを作成して別のVMに復元したり、他のリージョンにコピーしたりする用途に最も適した方法はどれか。

  1. A. Azure Backupによるバックアップ：Recovery Services コンテナーにVMバックアップポリシーを設定する
  2. B. ディスクのエクスポート（SAS URL）：ディスクの内容をVHDファイルとしてストレージアカウントにダウンロードする
  3. C. Azure Site Recovery：プライマリリージョンからセカンダリリージョンへ継続的にVMを複製する
  4. D. マネージドディスクのスナップショット：特定時点のディスク全体の読み取り専用コピーを作成し、新しいディスクや別VMの作成に使用できる
解答と解説を見る

正解: D

マネージドディスクのスナップショットは、任意の時点でディスク全体の読み取り専用コピーをAzure内で作成する機能であり、そのスナップショットから新しいManaged Diskを作成して別のVMにアタッチしたり、別リージョンのストレージにコピーしたりする操作が容易にできる。クイックな一時点復元や開発環境への複製に最適。選択肢AのAzure Backupはポリシーベースの定期バックアップであり、即時の任意時点スナップショット作成や別VMへの直接アタッチには向かない。選択肢BのSASエクスポートは外部転送（VHDダウンロード）に使うが、Azure内での新規ディスク作成には遠回りで低速。選択肢CのSite RecoveryはDR（障害復旧）のための継続複製であり、単純なディスクバックアップとは異なる目的の機能。
