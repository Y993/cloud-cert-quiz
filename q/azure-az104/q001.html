AZ-104Azure の ID とガバナンスの管理EASY単一選択

ある企業が Microsoft Entra ID（旧 Azure Active Directory）で社員アカウントを管理している。新入社員 50 名分のユーザーアカウントを一括で作成する必要があるが、管理者の手作業を最小限に抑えたい。最もコスト効率よく運用負荷を低減できる方法はどれか。

  1. A. CSV ファイルを用意し、Entra ID の「一括ユーザー作成」機能でインポートする
  2. B. PowerShell スクリプトで New-AzureADUser を 50 回ループ実行し、各実行後に portal で確認する
  3. C. Azure portal で 1 件ずつ手動作成し、各ユーザーに初期パスワードをメール送信する
  4. D. Microsoft Graph API を使って個別にリクエストを 50 回送信し、レスポンスを CSV に記録する
解答と解説を見る

正解: A

Entra ID（Azure AD）の「一括ユーザー作成（Bulk create users）」機能は、所定のテンプレート CSV をアップロードするだけで複数ユーザーを一括登録でき、管理者の手作業を最小化できる。選択肢Cは 1 件ずつ操作が必要で 50 名分は非現実的な運用負荷になる。選択肢Bは PowerShell でループ実行できるが、50 回ループ後に portal で確認するというアプローチは冗長で非効率であり、スクリプトが完結していれば portal 確認は不要。選択肢Dも個別 API コールを 50 回送る設計は一括操作と比べて実装コストが高く、バッチ CSV の方がずっとシンプルに同じ結果を得られる。
