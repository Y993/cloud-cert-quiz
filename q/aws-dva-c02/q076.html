DVA-C02セキュリティMEDIUM単一選択

ある開発者は AWS Systems Manager Parameter Store を使って設定値を管理しています。現在、標準パラメータ（Standard tier）を使用していますが、8KB を超える設定ファイルを保存したいという新しい要件が出てきました。最小限の変更で要件を満たすにはどうすればよいですか？

  1. A. Parameter Store を諦めて Secrets Manager に移行し大容量のデータを保存する
  2. B. 標準パラメータのままで使用し、8KB を超える部分は複数のパラメータに分割して保存する
  3. C. パラメータを Advanced tier に変更する（Advanced パラメータは最大 8KB → 最大 8KB のため変更不要）
  4. D. パラメータを Advanced tier に変更する（Advanced パラメータは最大 8192 バイトまで格納可能）
解答と解説を見る

正解: D

Parameter Store の Standard tier は最大 4KB（4096 バイト）、Advanced tier は最大 8KB（8192 バイト）のパラメータを保存できます。8KB を超えるデータが要件の場合は Advanced tier への変更が必要です。Advanced tier はパラメータあたり月額 0.05 USD の追加費用が発生しますが、運用上の変更は最小です。B の分割保存はアプリケーション側で結合ロジックを実装する必要があり、管理負荷が増加します。C の説明は誤りで、Standard は 4KB、Advanced は 8KB が上限であり異なります。A の Secrets Manager への移行はコストが高く、設定ファイル管理には過剰です。
