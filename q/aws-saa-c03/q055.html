SAA-C03高パフォーマンスなアーキテクチャの設計HARD単一選択

ある企業が、オンプレミスのデータセンターから AWS へのデータ移行を計画している。移行対象は約 200TB のファイルデータで、現在のインターネット接続帯域幅は 1Gbps である。移行を 1 週間以内に完了させたい。最も適切な移行方法はどれか。

  1. A. AWS DataSync を使って 1Gbps のインターネット回線経由で 200TB を転送する。
  2. B. Amazon S3 Transfer Acceleration を有効化して S3 へのアップロード速度を向上させる。
  3. C. AWS Direct Connect の 1Gbps 専用線を新規に契約し、転送速度を最大化する。
  4. D. AWS Snowball Edge を使ってデータを物理デバイスに書き込み、AWS に返送して S3 にインポートする。
解答と解説を見る

正解: D

200TB を 1Gbps で転送すると、理論値で約 1,600,000 秒（18.5 日）かかる。1 週間（約 7 日）以内の要件を満たさない。AWS Snowball Edge は物理デバイス（80〜100TB 容量）に高速でデータを書き込み（最大 100Gbps のローカル接続）、デバイスを AWS に返送してインポートする。インターネット帯域幅に依存しないため、大量データの高速移行に最適。200TB には 2〜3 台の Snowball Edge が必要で、物流時間込みでも 1 週間以内に完了できる。選択肢AのDataSync は 1Gbps でも 200TB 転送に 18 日以上かかり、1 週間以内の要件を満たさない。選択肢CのDirect Connect 新規契約は開通まで数週間〜数ヶ月かかることが多く、1 週間以内には間に合わない。選択肢BのTransfer Acceleration はアップロード速度を向上させるが、元の帯域幅制限（1Gbps）がボトルネックとなるため、200TB を 1 週間以内に転送するには不十分。
