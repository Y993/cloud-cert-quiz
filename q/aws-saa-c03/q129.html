SAA-C03コストを最適化したアーキテクチャの設計MEDIUM単一選択

ある企業が、AWS の本番環境のアーキテクチャを見直した結果、インターネットに公開しているサービスのデータ転送アウトバウンドコストが高額であることが判明した。月あたり約 50TB のデータをユーザーに配信している。最もコスト効率よくデータ転送コストを削減する方法はどれか。

  1. A. EC2 インスタンスを複数のリージョンに分散配置して各リージョンの転送量を削減する。
  2. B. Amazon S3 Transfer Acceleration を有効化してデータ転送を最適化する。
  3. C. Amazon CloudFront を経由してコンテンツを配信する。CloudFront のデータ転送単価は S3 や EC2 からの直接配信より安く、キャッシュヒット率が高いほど全体の転送量も削減できる。
  4. D. AWS Direct Connect を追加してインターネット経由の転送コストを削減する。
解答と解説を見る

正解: C

Amazon CloudFront のデータ転送アウトバウンドコストは、S3 や EC2 からの直接インターネット転送コストより安い。特に CloudFront のキャッシュヒット率が高い場合、実際にオリジン（S3/EC2）からデータを取得する量が削減されるため、オリジンのデータ転送コストも下がる。50TB/月のような大量転送では CloudFront の節約効果が大きい。選択肢BのS3 Transfer Acceleration はアップロード（S3 へのデータ書き込み）の高速化であり、コンテンツ配信（ダウンロード）コストの削減には直接貢献しない。選択肢AのEC2 マルチリージョン配置はインフラコストが増加し、各リージョンの転送量は分散しても総転送コストは変わらない。選択肢DのDirect Connect はオンプレミス接続向けであり、インターネットユーザーへのデータ配信コストの削減にはならない。
