SAA-C03コストを最適化したアーキテクチャの設計HARD単一選択

ある企業が、AWS のマルチアカウント環境で各部門のコストを把握し、部門別のコスト配分レポートを経営層に提出したい。各リソースが確実に部門に紐付けられ、タグが付いていないリソースのコストも分類できるようにしたい。最も適切なアプローチはどれか。

  1. A. Amazon QuickSight で CUR（コストと使用状況レポート）を可視化して部門別ダッシュボードを作成する。
  2. B. AWS Organizations でメンバーアカウントを部門ごとの OU（組織単位）に整理し、AWS コスト配分タグを有効化してリソースに部門タグを付与する。AWS Cost Explorer のタグベースのコストグループで部門別コストを集計する。タグがないリソースは AWS Organizations の一括請求（Consolidated Billing）でアカウント単位に集計する。
  3. C. AWS Cost Explorer で全アカウントのコストを可視化し、月次で手動レポートを作成する。
  4. D. 各部門に別々の AWS アカウントを作成し、請求書を部門ごとに分離する。
解答と解説を見る

正解: B

部門別コスト管理のベストプラクティスは、Organizations の OU 構成でアカウントを部門ごとに整理し、コスト配分タグで詳細な分類を行うことである。タグが付いていないリソースはアカウント単位（OU ベース）で部門に紐付けられるため、未タグリソースのコストも部門に帰属させられる。Cost Explorer のタグベースコストグループや AWS Cost Categories で細かい集計ルールを定義できる。選択肢Cは手動レポートで運用コストが高く、スケーラブルではない。選択肢Dは各部門に別アカウントを作成するだけでは、複数部門が共有するサービス（共通インフラ等）のコスト配分ができない。また既存のアカウント構成を変更する大規模な移行が必要になる。選択肢AのQuickSight + CUR は有効な可視化手段だが、コストの「分類・配分」の仕組み（タグ・アカウント設計）の構築が前提であり、それ自体では部門別コスト配分の問題を解決しない。
