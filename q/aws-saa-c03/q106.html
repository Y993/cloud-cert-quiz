SAA-C03コストを最適化したアーキテクチャの設計EASY単一選択

ある企業が、AWS の予算管理を強化したい。特定のサービスコストが月次予算の 80% に達した時点でアラートを受け取り、コスト超過を事前に防ぎたい。最小限の設定で実現できる最も適切な方法はどれか。

  1. A. Amazon CloudWatch でカスタムメトリクスを作成してコストを追跡し、アラームを設定する。
  2. B. AWS Cost Explorer で毎日コストを手動確認し、予算に近づいたら担当者にメールを送る。
  3. C. AWS Budgets で月次コスト予算を作成し、予算の 80% に達した際に SNS 経由でメール通知するアラートを設定する。
  4. D. AWS Trusted Advisor でコスト超過の可能性があるリソースを定期的に確認する。
解答と解説を見る

正解: C

AWS Budgets は月次・四半期・年次のコスト・使用量・Reserved Instances・Savings Plans のカバレッジ予算を定義し、実際コストが予算のX%に達した際に SNS / Email で通知するアラートを設定できる。コンソールから数分で設定でき、特定のサービス・アカウント・タグ単位での予算設定も可能。最初の 2 個の予算は無料で提供される。選択肢Bの手動確認は自動化されておらず、担当者の不在時やリアルタイムの検知ができない。選択肢AのCloudWatch カスタムメトリクスによるコスト追跡は可能だが、AWS Budgets のように専用に設計されておらず、設定が複雑で「最小限の設定」要件に合わない。選択肢DのTrusted Advisor はコスト最適化の推奨事項を提供するが、カスタムの月次予算に対するリアルタイムアラートの機能は提供しない。
