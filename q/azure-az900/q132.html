AZ-900AzureのアーキテクチャとサービスMEDIUM単一選択

ある企業がAzureで仮想マシンを管理しており、複数のVMに同一の設定（ソフトウェアインストール・OS設定・監視エージェント導入）を一括で適用し、その後も継続的に設定のドリフトを自動修正したい。最も適切なAzureサービスはどれか。

  1. A. Azure Automation State Configuration（DSC）：PowerShell DSCを使ってVM設定を宣言的に定義し、継続的に強制適用する構成管理サービス
  2. B. Azure Monitor：VMのメトリクスとログ監視サービス
  3. C. Azure App Service：WebアプリのPaaSホスティング
  4. D. Azure Virtual Machine Scale Sets：VMの自動スケーリングサービス
解答と解説を見る

正解: A

Azure Automation State Configuration（Desired State Configuration/DSC）はPowerShell DSCを使ってVMの望ましい状態（インストールするソフトウェア・サービスの起動状態・OS設定など）を宣言的に定義し、Automation Accountに登録されたVMに継続的に設定を適用・強制する構成管理サービスである。設定のドリフト（手動変更による設定のずれ）を自動的に検出・修正できる。選択肢DのVMSSはVM数の自動スケーリング機能であり、個別VMの詳細設定管理は担当しない（新規インスタンスはイメージベースの初期設定のみ）。選択肢BAzure MonitorはKPIモニタリング・ログ収集・アラートであり、設定の一括適用・ドリフト修正機能はない。選択肢CのApp ServiceはPaaSのWebアプリホスティングでVM構成管理とは無関係。
