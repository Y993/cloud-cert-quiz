AZ-900AzureのアーキテクチャとサービスMEDIUM単一選択

ある企業がAzureに移行後、オンプレミスのVMware・Hyper-V・物理サーバー・AWSなどの他クラウドにあるリソースも含めて、一つのAzureコンソールから一元管理・ガバナンスポリシー適用・セキュリティ監視を行いたい。最も適切なAzureサービスはどれか。

  1. A. Azure Migrate：オンプレミスからAzureへの移行を支援するサービス
  2. B. Azure Site Recovery：ディザスタリカバリーのためのレプリケーションサービス
  3. C. Azure Arc：Azure外のリソース（オンプレミスVM・他クラウドVM・Kubernetesクラスター）をAzureの管理ツールに統合するハイブリッド管理サービス
  4. D. Azure Virtual Desktop：リモートデスクトップ仮想化サービス
解答と解説を見る

正解: C

Azure ArcはAzure外のリソース（オンプレミスのWindows/Linux VM・VMware・Hyper-V・AWS・Google Cloud上のVM・Kubernetesクラスター・SQLサーバー）にAzure Arcエージェントをインストールすることで、Azureポータルからの一元管理・Azure PolicyやRBACの適用・Defender for Cloudのセキュリティ監視・Azure Monitorでのログ収集を実現するハイブリッド管理サービスである。選択肢AAzure MigrateはオンプレミスからAzureへの初期移行（アセスメント・移行ツール）に特化しており、継続的なマルチクラウド管理には使用しない。選択肢DAzure Virtual DesktopはWindows仮想デスクトップのVDIサービスであり、マルチクラウド/ハイブリッド管理とは異なる。選択肢BAzure Site RecoveryはDR（ディザスタリカバリー）のためのレプリケーションサービスであり、マルチクラウドの統合管理機能はない。
