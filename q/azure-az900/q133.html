AZ-900Azureの管理とガバナンスMEDIUM単一選択

ある企業がAzureのコストを最適化するために、開発チームが業務外時間（夜間・週末）に使用しないAzure VMを自動的に停止・起動したい。最も適切なAzureサービスはどれか。

  1. A. Azure Automation（Runbook）またはAzure VM Auto-shutdown：スケジュールに基づいてVMを自動起動・停止する
  2. B. Azure Monitor：VMのメトリクスを監視してアラートを送信する
  3. C. Azure Cost Management：コストを追跡して予算アラートを設定する
  4. D. Azure Advisor：コスト最適化の推奨事項を提供する
解答と解説を見る

正解: A

Azure AutomationのRunbook（スクリプトベースの自動化）またはAzure VMの組み込み「Auto-shutdown」機能を使うと、指定した時刻（例：平日20:00）に自動シャットダウンのスケジュールを設定できる。起動については、Azure AutomationのスケジュールRunbookを使って指定時刻（例：平日8:00）に自動的にVMを起動する処理を実行できる。選択肢BAzure Monitorはパフォーマンス監視とアラートであり、VMの自動起動・停止スケジューリング機能はない（MonitorのアクションからRunbookを呼び出すことは可能）。選択肢CAzure Cost Managementはコスト追跡・予算アラートであり、VM起動・停止の自動化機能はない。選択肢DAzure Advisorは推奨事項の提示（「未使用VMを停止してください」など）はするが、自動起動・停止を実行する機能はない。
