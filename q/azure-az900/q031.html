AZ-900AzureのアーキテクチャとサービスMEDIUM単一選択

ある企業がAzureの仮想マシン（VM）を使ってWebサーバーをデプロイした。同一リージョン内で複数のVMにトラフィックを均等に分散し、特定のVMが応答しなくなった場合は自動的にそのVMへのトラフィックを停止したい。最も適切なAzureサービスはどれか。

  1. A. Azure CDN：静的コンテンツをエッジキャッシュから配信する
  2. B. Azure VPN Gateway：オンプレミスとAzureをVPN接続するゲートウェイ
  3. C. Azure Firewall：ネットワークトラフィックのフィルタリングとセキュリティポリシーを適用する
  4. D. Azure Load Balancer：レイヤー4でトラフィックを分散し、ヘルスプローブで障害VMを検出する
解答と解説を見る

正解: D

Azure Load Balancerはレイヤー4（TCP/UDP）でトラフィックを複数のバックエンドVMに均等分散し、ヘルスプローブ（定期的な死活監視）により応答しないVMを自動的にローテーションから外す機能を持つ。同一リージョン内のVMへの内部・外部負荷分散に使用される。選択肢BのVPN GatewayはオンプレミスとAzure間のVPN接続確立に使うサービスであり、VM間のトラフィック分散には使用しない。選択肢CAzure Firewallはネットワーク境界のセキュリティ制御（L3-L7フィルタリング）を行うサービスで、負荷分散機能はない。選択肢AのCDNは静的コンテンツのエッジキャッシングサービスであり、バックエンドVMへのトラフィック分散には使用しない。
